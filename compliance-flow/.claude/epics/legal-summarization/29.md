# Task: Workflow execution integration and testing

## Description
Wire the Document Node into the workflow execution engine so document text and summaries flow to downstream nodes. Ensure Output Node handles document summaries correctly for all output types. Perform end-to-end testing of the full pipeline.

## Acceptance Criteria
- [ ] `documentNode` case added to `workflowStore.ts` `runWorkflow()`:
  - Reads document from documentStore by node config's document ID
  - If mode is 'summarize': triggers summarization, stores result in `workflowData.documentText` and `workflowData.documentSummary`
  - If mode is 'batch': processes all documents, stores `workflowData.batchSummaries`
  - If mode is 'search': stores search query and results in `workflowData.searchResults`
  - Adds execution logs: "Parsing document...", "Summarizing with template: {name}...", "Summary complete"
- [ ] Downstream LLM nodes receive document text/summary as context (in prompt building section)
- [ ] Downstream PII Filter nodes can filter summary text
- [ ] Output Node handles document summaries:
  - **Chat**: displays formatted summary with section headers
  - **Spreadsheet**: one row per document (batch mode), columns per template field
  - **Email**: summary as formatted body
  - **Telegram**: summary as message text
- [ ] `workflowData` context building updated to include `documentText`, `documentSummary`, `batchSummaries`, `searchResults`
- [ ] End-to-end workflow: Trigger → Document → LLM → PII Filter → Output (chat) works completely
- [ ] End-to-end workflow: Trigger → Document (batch) → Output (spreadsheet) exports correctly

## Technical Details
- Follow exact pattern from existing node cases in `runWorkflow()` (see dockerContainerNode and llmNode cases)
- Document Node execution steps:
  1. Check if document is uploaded (warn if not)
  2. Get extracted text from documentStore
  3. If summarize mode: call `summarizeDocument()` from summarizationService
  4. Set `workflowData.documentText` and `workflowData.documentSummary`
  5. Add logs at each step
- LLM context building (line ~296-313 in workflowStore.ts): add `workflowData.documentText` and `workflowData.documentSummary` to contextParts
- PII filter: apply to `workflowData.documentSummary` text (join all field values)
- Spreadsheet output: transform `batchSummaries` into rows where each field becomes a column
- Files affected: `src/store/workflowStore.ts` (edit — add documentNode case + context building), `src/components/panels/NodeConfigPanel.tsx` (minor — ensure run button works)
