# Task: Types and document store

## Description
Define all TypeScript interfaces for the legal summarization feature and create the Zustand document store with 6 default extraction templates.

## Acceptance Criteria
- [ ] `src/types/document.ts` created with interfaces: `DocumentConfig`, `ExtractionTemplate`, `TemplateField`, `DocumentMeta`, `DocumentSummary`, `SummaryField`, `EvaluationResult`, `SearchEntry`
- [ ] `src/types/index.ts` updated to re-export document types
- [ ] `src/store/documentStore.ts` created following existing Zustand persist pattern (`create<State>()(persist(...))`)
- [ ] Store includes state: `templates`, `documents`, `summaries`, `searchIndex`, `evaluationResults`
- [ ] Store includes actions: `addTemplate`, `updateTemplate`, `deleteTemplate`, `addDocument`, `updateDocument`, `addSummary`, `addEvaluationResult`, `addSearchEntry`, `clearSearchIndex`
- [ ] 6 default extraction templates included: NDA, Lease/Sublease, Service Agreement, Employment Contract, Court Filing, Regulatory Report
- [ ] Each template has named fields with descriptions and extraction instructions
- [ ] Templates support import/export as JSON
- [ ] Store persists to localStorage as `compliance-document-storage`

## Technical Details
- Follow pattern from `src/store/dockerStore.ts` and `src/store/workflowStore.ts`
- `ExtractionTemplate` shape: `{ id, name, description, documentType, systemPrompt, fields: TemplateField[] }`
- `TemplateField` shape: `{ name, description, instructions }`
- `DocumentMeta` shape: `{ id, name, fileType, fileSize, pageCount, extractedText, uploadedAt, status }`
- `DocumentSummary` shape: `{ id, documentId, templateId, fields: SummaryField[], rawOutput, createdAt, chunkSummaries? }`
- Persisted state should exclude `extractedText` content (too large for localStorage) — only store metadata
- Files affected: `src/types/document.ts` (new), `src/types/index.ts` (edit), `src/store/documentStore.ts` (new)
