# Task: Backend document parsing service

## Description
Add Python backend service for parsing PDF and DOCX files into clean text. Create API routes for file upload and text extraction.

## Acceptance Criteria
- [ ] `backend/app/services/document_service.py` created with PDF and DOCX parsing
- [ ] PDF parsing using PyPDF2: extracts text from all pages, removes extra whitespace, strips page numbers
- [ ] DOCX parsing using python-docx: extracts paragraphs, headings, and table text
- [ ] TXT handling: reads plain text with encoding detection
- [ ] Text cleaning: whitespace normalization, header/footer stripping
- [ ] Metadata extraction: page count, file size, detected file type
- [ ] `backend/app/api/documents.py` created with `POST /api/v1/documents/parse` route
- [ ] Parse endpoint accepts multipart file upload (PDF, DOCX, TXT)
- [ ] Returns JSON: `{ success, text, metadata: { pages, size, type }, error? }`
- [ ] Graceful handling of malformed files (clear error message, no crash)
- [ ] `backend/requirements.txt` updated with PyPDF2, python-docx
- [ ] Route registered in `backend/app/main.py`

## Technical Details
- Follow pattern from existing `backend/app/api/databases.py` and `backend/app/services/database.py`
- Use FastAPI `UploadFile` for file handling
- Max file size: 100MB (configurable)
- Text cleaning regex: remove `\s+` → single space, remove standalone page numbers
- Return extracted text as a single string (frontend handles chunking)
- Files affected: `backend/app/services/document_service.py` (new), `backend/app/api/documents.py` (new), `backend/requirements.txt` (edit), `backend/app/main.py` (edit)
