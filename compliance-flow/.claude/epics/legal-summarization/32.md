# Task: Document Node and canvas integration

## Description
Create the Document Node component for the workflow canvas and integrate it into the sidebar, node registry, and config panel.

## Acceptance Criteria
- [ ] `src/components/nodes/DocumentNode.tsx` created wrapping BaseNode with FileText icon
- [ ] Node displays: document name (or "No document"), file type badge, template name, mode indicator (summarize/search/batch)
- [ ] Color: `bg-gradient-to-br from-[#4004DA] to-[#6B3FA0]` (matching AI/compliance theme)
- [ ] Registered as `documentNode` in `src/components/nodes/index.ts`
- [ ] Added to Sidebar under new "Documents" category (between "Data Sources" and "AI Models")
- [ ] Sidebar entry: label "Legal Document", icon FileText, color `bg-[#4004DA]`
- [ ] `DocumentNodeConfig` added to `NodeConfigPanel.tsx` with:
  - Document upload zone (using DocumentUploadZone component)
  - Template selector dropdown (populated from documentStore)
  - Mode toggle: Summarize / Search / Batch
  - Chunk size input (default 20000, shown only in Summarize mode)
  - System prompt override textarea
  - Save Configuration button
- [ ] NodeIcon case added for `documentNode` in NodeConfigPanel
- [ ] Default node config: `{ mode: 'summarize', templateId: null, chunkSize: 20000, documents: [] }`

## Technical Details
- Follow exact pattern from `LLMNode.tsx` and `DockerContainerNode.tsx`
- Import `FileText` from lucide-react for icon
- Template selector: use `Select` common component, options from `useDocumentStore((s) => s.templates)`
- Upload zone: embed `DocumentUploadZone` component from Task #30
- When file is uploaded, call backend `/api/v1/documents/parse` to extract text, store in documentStore
- Mode toggle: three buttons styled like PII Filter mode selector
- Files affected: `src/components/nodes/DocumentNode.tsx` (new), `src/components/nodes/index.ts` (edit), `src/components/sidebar/Sidebar.tsx` (edit), `src/components/panels/NodeConfigPanel.tsx` (edit)
