# Task: Evaluation dashboard

## Description
Add backend evaluation endpoints for ROUGE/BLEU scoring and create a frontend evaluation panel with quality metrics, threshold configuration, and LLM-based grading.

## Acceptance Criteria
- [ ] `backend/app/services/evaluation_service.py` created with:
  - `calculate_rouge(generated, reference)` → ROUGE-1, ROUGE-2, ROUGE-L F1 scores
  - `calculate_bleu(generated, reference)` → BLEU score
  - `calculate_similarity(embedding1, embedding2)` → cosine similarity
- [ ] `POST /api/v1/documents/evaluate` backend endpoint:
  - Accepts: `{ generated_summary, reference_summary }`
  - Returns: `{ rouge_1, rouge_2, rouge_l, bleu, similarity? }`
- [ ] `backend/requirements.txt` updated with `rouge-score`, `nltk`
- [ ] `src/components/panels/EvaluationPanel.tsx` created:
  - Floating, resizable panel (similar to ChatInterfacePanel pattern)
  - Reference summary upload: text input or file upload for expert-written reference
  - "Evaluate" button triggers scoring against selected document summary
  - Per-document scores table: Document Name, ROUGE-1, ROUGE-2, ROUGE-L, BLEU, Pass/Fail
  - Aggregate metrics: average scores across all evaluated documents
  - Quality threshold configuration: sliders for minimum acceptable scores
  - Visual pass/fail indicators (green check / red X) based on thresholds
- [ ] LLM-based grading option:
  - "Grade with AI" button sends summary + rubric to Ollama
  - Rubric: accuracy (1-5), completeness (1-5), legal precision (1-5), conciseness (1-5)
  - Displays LLM grade alongside automated metrics
- [ ] Evaluation results stored in `documentStore.evaluationResults`
- [ ] Panel accessible from Document Node config ("Evaluate Quality" button)

## Technical Details
- Backend: `rouge-score` Python library for ROUGE metrics, `nltk` for tokenization/BLEU
- Evaluation panel follows pattern from `ChatInterfacePanel.tsx` (floating, draggable, resizable)
- LLM grading prompt: "Rate the following summary on a scale of 1-5 for: accuracy, completeness, legal precision, conciseness. Provide scores in XML format: <accuracy>N</accuracy>..."
- Reuse XML parser from summarization service for LLM grade extraction
- Threshold defaults: ROUGE-L ≥ 0.35, BLEU ≥ 0.20
- Files affected: `backend/app/services/evaluation_service.py` (new), `backend/app/api/documents.py` (edit — add /evaluate route), `backend/requirements.txt` (edit), `src/components/panels/EvaluationPanel.tsx` (new), `src/components/panels/NodeConfigPanel.tsx` (edit — add "Evaluate Quality" button)
