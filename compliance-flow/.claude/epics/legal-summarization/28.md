# Task: Structured summarization engine

## Description
Build the core summarization logic: prompt construction from extraction templates, XML-tagged output parsing, and integration with the existing Ollama `/llm/generate` API. Add a summary preview display in the Document Node config panel.

## Acceptance Criteria
- [ ] `src/services/summarizationService.ts` created with:
  - `buildSummarizationPrompt(text, template)` — constructs prompt from template fields using XML-tagged format per Claude legal summarization guide
  - `parseSummaryOutput(output, template)` — parses XML-tagged LLM output into structured `SummaryField[]`
  - `summarizeDocument(documentId, templateId)` — orchestrates: get text → build prompt → call API → parse output → store summary
- [ ] Prompt format follows Claude guide pattern:
  ```
  Summarize the following {documentType}. Focus on these key aspects:
  {template fields with descriptions}

  Provide the summary in bullet points nested within the XML header for each section.
  <{field_name}>
  - Key detail: [extracted value]
  </{field_name}>

  If not found, note as "Not specified".
  ```
- [ ] System prompt configurable per template (e.g., "You are a legal analyst specializing in real estate law")
- [ ] XML parser extracts content between `<fieldName>` and `</fieldName>` tags
- [ ] Graceful fallback: if XML parsing fails, store raw output as single field
- [ ] Temperature set to 0.2 for legal precision (overrides node default)
- [ ] Summary stored in documentStore via `addSummary()`
- [ ] Config panel shows "Summarize" button that triggers summarization
- [ ] Summary preview panel shows parsed fields with section headers
- [ ] Loading state during summarization (spinner + "Summarizing..." text)

## Technical Details
- Use existing `api.generate()` from `src/services/api.ts` for LLM calls
- XML parsing: regex-based extraction `/<(\w+)>([\s\S]*?)<\/\1>/g` with fallback
- Prompt construction: iterate `template.fields`, join with newlines, wrap in instruction format
- Summary preview: render each `SummaryField` as a collapsible section with field name as header
- Add "Summarize" button to DocumentNodeConfig (visible when mode === 'summarize' and document is uploaded)
- Files affected: `src/services/summarizationService.ts` (new), `src/components/panels/NodeConfigPanel.tsx` (edit — add summary preview and trigger button to DocumentNodeConfig)
