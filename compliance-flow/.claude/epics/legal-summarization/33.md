# Task: Batch processing

## Description
Extend the Document Node to accept multiple files and process them sequentially with progress tracking, error isolation, and aggregated results for downstream nodes.

## Acceptance Criteria
- [ ] Document Node config in "Batch" mode shows multi-file upload zone
- [ ] DocumentUploadZone used with `multiple={true}` prop
- [ ] Batch processing queue: files processed one-at-a-time sequentially
- [ ] Per-document status tracking: pending, parsing, summarizing, complete, error
- [ ] Batch progress UI: progress bar, "Processing 3/20..." counter, per-doc status list
- [ ] Error isolation: failed documents logged and skipped, batch continues
- [ ] Failed documents shown with error details (red indicator + message)
- [ ] Batch results aggregated: `workflowData.batchSummaries = DocumentSummary[]`
- [ ] "Cancel Batch" button to stop processing remaining documents
- [ ] Batch completion summary: X succeeded, Y failed, Z total

## Technical Details
- Batch mode reuses `summarizeDocument()` from Task #28 per file
- Processing loop: parse file → summarize → store result → next file
- Use `documentStore` to track per-document status during batch
- Progress state managed in component (not store — ephemeral UI state)
- Batch results stored as array of DocumentSummary in documentStore
- Cancel via AbortController or boolean flag checked between documents
- Downstream integration: Output Node spreadsheet mode can iterate `batchSummaries` to create one row per document
- Files affected: `src/components/panels/NodeConfigPanel.tsx` (edit — batch mode UI in DocumentNodeConfig), `src/services/summarizationService.ts` (edit — add `summarizeBatch()` function)

- [ ] Task #31: Meta-summarization (long docs in batch need chunking)
