# Task: Summary-indexed document search (RAG)

## Description
Add document search capability using summary-indexed RAG. Documents are indexed by generating embeddings of their summaries via Ollama. Users query in natural language and get ranked results with relevance scores and excerpts.

## Acceptance Criteria
- [ ] Backend `POST /api/v1/documents/embed` endpoint using Ollama embeddings API
  - Accepts text string, returns embedding vector
  - Uses Ollama model `nomic-embed-text` (or configurable)
- [ ] `backend/app/api/documents.py` updated with `/embed` route
- [ ] Document indexing: when a document is summarized, its summary is also embedded and stored in `documentStore.searchIndex`
- [ ] `SearchEntry` stored: `{ documentId, documentName, summaryText, embedding: number[], indexedAt }`
- [ ] Search mode in DocumentNodeConfig:
  - Natural language query input field
  - "Search" button
  - Results list showing: document name, relevance score (%), matching excerpt
- [ ] Cosine similarity computed client-side: `cosineSimilarity(queryEmbedding, docEmbedding)`
- [ ] Results ranked by similarity score, top 10 displayed
- [ ] Each result clickable to view full summary
- [ ] Search results wired to `workflowData.searchResults` for downstream nodes
- [ ] Chat panel integration: search results displayed as formatted messages with document citations

## Technical Details
- Ollama embeddings endpoint: `POST http://localhost:11434/api/embeddings` with `{ model, prompt }`
- Backend `/embed` proxies to Ollama embeddings API (similar to existing `/llm/generate` proxy)
- Cosine similarity: `dot(a,b) / (norm(a) * norm(b))` — pure JS, no library needed
- Query flow: user enters query → embed query via backend → compare against all indexed docs → rank by similarity → display top results
- Embedding vectors typically 768-1024 dimensions, stored as `number[]` in Zustand store
- Note: searchIndex excluded from localStorage persistence (too large) — rebuilt on demand
- Files affected: `backend/app/api/documents.py` (edit — add /embed route), `src/services/summarizationService.ts` (edit — add indexing after summarization), `src/store/documentStore.ts` (edit — add search index management), `src/components/panels/NodeConfigPanel.tsx` (edit — add search mode UI)
