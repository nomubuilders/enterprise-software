# Task: Document upload UI component

## Description
Create a reusable drag-and-drop file upload component that supports PDF, DOCX, and TXT files. Shows file info, upload progress, and parsing status.

## Acceptance Criteria
- [ ] `src/components/common/DocumentUploadZone.tsx` created
- [ ] Drag-and-drop zone with visual feedback (border highlight on drag-over)
- [ ] Click-to-browse file picker as fallback
- [ ] Accepts PDF, DOCX, TXT files (rejects others with error message)
- [ ] Single-file and multi-file modes (controlled via prop)
- [ ] Displays file info after upload: name, size, type icon
- [ ] Status indicators: idle, uploading, parsing, parsed, error
- [ ] Callback props: `onFileSelect(files)`, `onFileRemove(id)`
- [ ] Follows Nomu brand styling: Tailwind utilities, CSS variables, Barlow headings
- [ ] File type icons from lucide-react (FileText for PDF, FileType for DOCX, File for TXT)
- [ ] Exported from `src/components/common/index.ts`

## Technical Details
- Use native HTML5 drag-and-drop API (no external library)
- Accept MIME types: `application/pdf`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`, `text/plain`
- Max file size validation: 100MB per file
- Component props interface: `{ multiple?: boolean, onFileSelect: (files: File[]) => void, onFileRemove?: (index: number) => void, files?: File[], status?: string, accept?: string }`
- Follow existing component patterns in `src/components/common/` (Button, Input, Modal, Select)
- Files affected: `src/components/common/DocumentUploadZone.tsx` (new), `src/components/common/index.ts` (edit)

## Dependencies
