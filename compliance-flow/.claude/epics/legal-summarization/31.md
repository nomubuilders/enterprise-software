# Task: Meta-summarization and chunking

## Description
Implement document chunking for long documents that exceed the LLM context window. Each chunk is summarized independently, then chunk summaries are combined into a final meta-summary. Add UI for chunk inspection.

## Acceptance Criteria
- [ ] `chunkText(text, chunkSize, overlap)` function in summarizationService.ts
  - Splits text at sentence boundaries (not mid-word/sentence)
  - Default chunk size: 20,000 characters
  - Default overlap: 500 characters
  - Returns array of `{ text, startIndex, endIndex }` chunks
- [ ] Auto-detection: if document text > chunkSize, automatically use chunked summarization
- [ ] Each chunk summarized independently using same template via `summarizeDocument()`
- [ ] Meta-summarization: chunk summaries aggregated and sent as final prompt:
  ```
  You are looking at chunked summaries of a {documentType}.
  Combine the following summaries into a coherent overall summary:
  {chunk summaries}
  Focus on: {template fields}
  ```
- [ ] Individual chunk summaries stored in `DocumentSummary.chunkSummaries[]`
- [ ] Progress tracking: "Summarizing chunk 2/5..." status updates
- [ ] Expandable debug view in config panel showing per-chunk summaries
- [ ] Chunk size configurable per node (already in DocumentNodeConfig from Task #32)

## Technical Details
- Sentence boundary detection: split on `.!?` followed by whitespace and uppercase letter
- If sentence splitting produces chunks > 2x target size, fall back to paragraph splitting (`\n\n`)
- Sequential chunk processing (not parallel — avoid overwhelming Ollama)
- Progress callback pattern: `onProgress(current, total, status)` updates UI state
- Debug view: collapsible accordion with chunk number, char range, and summary text
- Meta-summary prompt reuses same system prompt as individual chunks
- Files affected: `src/services/summarizationService.ts` (edit — add chunking), `src/components/panels/NodeConfigPanel.tsx` (edit — add chunk debug view)
