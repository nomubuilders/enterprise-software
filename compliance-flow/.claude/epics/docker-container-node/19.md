# Task: Implement image allowlist enforcement and Cosign verification

## Description
Add pre-execution security checks: verify the requested image is on the admin-curated allowlist before execution, and optionally verify the image signature using Cosign. Block execution with a clear error if any check fails, and log blocked attempts as security events.

## Acceptance Criteria
- [ ] Pre-execution middleware checks image name:tag against `config/approved-images.json`
- [ ] If image not on allowlist: execution blocked, clear error message returned to frontend ("Image not approved: {image}. Contact your admin.")
- [ ] If image on allowlist: verify SHA256 hash matches (if sha256 field is set in allowlist)
- [ ] Per-image resource ceilings: if allowlist entry has maxCpu/maxMemory, user config cannot exceed those limits
- [ ] Cosign verification: if `cosign` binary is available on PATH, verify image signature before execution. If Cosign not available, log a warning but allow execution (graceful degradation)
- [ ] Blocked executions logged as security events with reason, attempted image, user ID, timestamp
- [ ] Frontend displays clear error UI when execution is blocked by security checks

## Technical Details
- Allowlist check is a synchronous function that reads approved-images.json and does exact match on `name:tag`
- SHA256 verification: compare `docker inspect --format='{{.Id}}'` output against allowlist sha256
- Cosign: shell out to `cosign verify --key {keypath} {image}` — parse exit code
- Resource ceiling: `Math.min(userConfig.cpu, allowlistEntry.maxCpu)` pattern
- Security event log: append to `data/audit/security-events.jsonl`
