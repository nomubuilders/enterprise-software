# Task: Add real-time log streaming and xterm.js terminal

## Description
Implement real-time container log streaming from the backend to the frontend using Server-Sent Events (SSE), and render logs in an xterm.js terminal emulator embedded in the NodeConfigPanel's Docker section.

## Acceptance Criteria
- [ ] Backend streams container stdout/stderr via SSE endpoint (`GET /api/v1/docker/logs/:containerId`)
- [ ] Frontend subscribes to SSE stream when container execution starts
- [ ] xterm.js terminal component renders in the Docker config panel as a collapsible "Execution Console" section
- [ ] Terminal shows real-time output with proper ANSI color support
- [ ] Terminal auto-scrolls to latest output
- [ ] Terminal is read-only (no user input)
- [ ] xterm.js is lazy-loaded (dynamic import) to avoid bloating main bundle
- [ ] Terminal renders correctly in both dark and light themes (background matches `--nomu-bg`, text matches `--nomu-text`)
- [ ] "Clear" button to reset terminal content

## Technical Details
- Install `xterm` and `xterm-addon-fit` npm packages
- Create `src/components/panels/DockerTerminal.tsx` — lazy-loaded wrapper around xterm.js
- SSE endpoint: backend uses `res.setHeader('Content-Type', 'text/event-stream')` and pipes Dockerode log stream
- Frontend: `new EventSource('/api/v1/docker/logs/${containerId}')` with `onmessage` handler writing to xterm
- Lazy load: `const DockerTerminal = React.lazy(() => import('./DockerTerminal'))`
- Theme: set xterm theme options `{ background: 'var(--nomu-bg)', foreground: 'var(--nomu-text)' }` — may need to resolve CSS vars to hex at runtime
