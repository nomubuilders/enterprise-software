# Task: Security hardening and resource watchdog

## Description
Apply defense-in-depth security measures to container execution: Seccomp profiles, non-root enforcement, privilege restrictions, and a watchdog process that monitors and kills containers exceeding resource limits or timeouts.

## Acceptance Criteria
- [ ] Containers run as non-root user by default (`User: '1000:1000'` in container config)
- [ ] Privileged mode explicitly disabled (`Privileged: false`)
- [ ] No host PID namespace sharing (`PidMode: ''`)
- [ ] No host IPC namespace sharing (`IpcMode: 'none'`)
- [ ] Default Seccomp profile applied (Docker's built-in default, or custom restrictive profile)
- [ ] Read-only root filesystem option (`ReadonlyRootfs: true`) with writable `/tmp` and `/output`
- [ ] Watchdog timer: background interval that checks all active containers against their timeout and kills any that exceed it
- [ ] Resource monitoring: log CPU/memory usage at container completion for audit trail
- [ ] No capability additions (`CapAdd: []`), drop all unnecessary caps (`CapDrop: ['ALL']`)

## Technical Details
- Dockerode HostConfig options: `Privileged`, `CapDrop`, `SecurityOpt`, `ReadonlyRootfs`, `PidMode`, `IpcMode`, `User`
- Seccomp: use `SecurityOpt: ['seccomp=unconfined']` only in dev, default profile in prod
- Watchdog: `setInterval` that iterates `dockerStore.activeContainers`, checks elapsed time vs timeout, calls `container.kill()` if exceeded
- Writable tmpfs mounts for `/tmp` and `/output`: `Tmpfs: { '/tmp': 'rw,noexec,size=100m', '/output': 'rw,noexec,size=500m' }`
