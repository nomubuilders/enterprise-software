# Task: Add Podman and remote Docker host support

## Description
Extend the Docker API service to support Podman (rootless containers) and remote Docker hosts via TCP/SSH, making ComplianceFlow deployable in stricter compliance environments that don't allow Docker daemon.

## Acceptance Criteria
- [ ] Configuration option for runtime: `docker` (default), `podman`, or `remote`
- [ ] Podman support: detect Podman socket at `/run/user/{uid}/podman/podman.sock` or `$XDG_RUNTIME_DIR/podman/podman.sock`
- [ ] Remote Docker: support `DOCKER_HOST` env var for TCP (`tcp://host:2376`) and SSH (`ssh://user@host`) connections
- [ ] TLS support for remote Docker: `DOCKER_TLS_VERIFY`, `DOCKER_CERT_PATH` env vars
- [ ] Health check correctly identifies which runtime is active
- [ ] All existing container lifecycle operations work with Podman and remote hosts
- [ ] Graceful error messages when selected runtime is unavailable

## Technical Details
- Dockerode supports custom socket paths: `new Dockerode({ socketPath: '/path/to/podman.sock' })`
- Dockerode supports remote hosts: `new Dockerode({ host: 'host', port: 2376, ca, cert, key })`
- Add runtime config to dockerStore or environment variables
- Runtime auto-detection order: check Docker socket → check Podman socket → check DOCKER_HOST env
- Podman API is Docker-compatible for most operations — minimal code changes needed
