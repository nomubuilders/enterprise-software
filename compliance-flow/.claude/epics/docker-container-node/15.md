# Task: Build Docker configuration section in NodeConfigPanel

## Description
Add a comprehensive Docker configuration section to the existing NodeConfigPanel component. When a dockerContainerNode is selected, the panel shows: image selector, command input, environment variables editor, volume mappings, resource control sliders, and network mode toggle.

## Acceptance Criteria
- [ ] **Image Selector**: Searchable dropdown populated from dockerStore's approvedImages. Shows image name, tag, and approval date. Displays "No approved images" message if allowlist is empty
- [ ] **Command Input**: Text input for command/entrypoint override with placeholder showing example (e.g., `python /app/scan.py`)
- [ ] **Environment Variables**: Dynamic key-value pair list with add/remove buttons. Keys are text inputs, values are text inputs with placeholder for expression syntax
- [ ] **Volume Mappings**: List of host path → container path pairs with add/remove. Input volume (`/input`) shown as auto-mounted (read-only display)
- [ ] **Resource Controls**: CPU slider (0.25-4.0, step 0.25, default 0.5), Memory slider (128-4096MB, step 128, default 512), Timeout input (30-3600s, default 300)
- [ ] **Network Mode Toggle**: Switch between "Air-Gapped" (network: none, default) and "Internal Only" (network: internal)
- [ ] All config changes update the node's data in flowStore
- [ ] Panel renders correctly in both dark and light themes using Nomu CSS variables

## Technical Details
- Add a new case for `dockerContainerNode` in NodeConfigPanel's node type switch
- Follow existing patterns for Database and LLM node configuration sections
- Use existing Input, Select, Button components from `src/components/common/`
- Resource sliders can use HTML range inputs styled with brand accent color
- Env var list: use a simple array of `{key: string, value: string}` objects in state
