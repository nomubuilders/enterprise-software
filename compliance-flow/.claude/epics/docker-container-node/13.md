# Task: Create Docker types, Zustand store, and node component

## Description
Create the foundation for the Docker Container Node: TypeScript interfaces, Zustand state store, and the canvas node component. This task establishes all shared types and state that other tasks depend on.

## Acceptance Criteria
- [ ] `src/types/docker.ts` created with interfaces: `DockerContainerConfig` (image, command, envVars, volumes, cpuLimit, memoryLimit, timeout, networkMode), `ApprovedImage` (name, tag, sha256, approvedBy, approvedAt, maxCpu, maxMemory), `ContainerExecution` (id, nodeId, status, image, startedAt, exitCode, logs, output), `ContainerAuditLog` (eventType, workflowId, workflowRunId, nodeId, userId, timestamp, image, imageSha256, command, resourceLimits, networkMode, exitCode, duration, outputSizeBytes)
- [ ] `src/store/dockerStore.ts` created with Zustand store managing: approvedImages array, activeContainers map, executionResults, actions for loadAllowlist, startExecution, updateExecution, completeExecution
- [ ] `src/components/nodes/DockerContainerNode.tsx` created using BaseNode wrapper with header color `bg-[#36312E]`, Container icon from lucide-react in purple, body showing image name/tag, resource limits summary, and air-gapped badge
- [ ] `src/types/workflow.ts` updated to export DockerContainerConfig
- [ ] All files compile clean with TypeScript strict mode

## Technical Details
- Follow existing patterns: `DatabaseConfig`/`LLMConfig` in `src/types/workflow.ts` for type style, `themeStore.ts` for Zustand pattern, `DatabaseNode.tsx` for node component pattern
- DockerContainerNode body should display: selected image name, CPU/RAM limits, network mode badge ("Air-Gapped" or "Internal Only")
- Store should use `persist` middleware for allowlist caching
- Use `Container` icon from lucide-react (or `Box` if Container not available)
