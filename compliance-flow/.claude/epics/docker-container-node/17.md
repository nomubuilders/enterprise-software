# Task: Create backend Docker API service and container orchestration

## Description
Create the Docker API service (`src/services/dockerApi.ts`) that communicates with the backend container orchestration layer. Implement the full container lifecycle: pull image, create container with config, start, wait for completion, capture output, and remove. Create the image allowlist config file.

## Acceptance Criteria
- [ ] `src/services/dockerApi.ts` created with methods: `checkDockerHealth()`, `listApprovedImages()`, `executeContainer(config: DockerContainerConfig)`, `streamLogs(containerId: string)`, `stopContainer(containerId: string)`, `getContainerStatus(containerId: string)`
- [ ] `config/approved-images.json` created with seed data: alpine:latest and python:3.11-slim as example approved images
- [ ] Container lifecycle implemented: pull → create → start → wait → capture output → remove
- [ ] Data injection: workflow input JSON mounted at `/input/data.json` via temp volume
- [ ] Output capture: reads `/output/result.json` from container, falls back to stdout JSON parsing
- [ ] Auto-cleanup: container always removed in finally block, even on error
- [ ] Timeout enforcement: container killed if execution exceeds configured timeout
- [ ] Docker health check endpoint to verify daemon availability
- [ ] API follows existing `src/services/api.ts` patterns (base URL, error handling)

## Technical Details
- Use Dockerode npm package for Docker Engine API communication
- Container create options: `NetworkDisabled` for air-gapped, `HostConfig.NanoCpus` for CPU, `HostConfig.Memory` for RAM
- Mount temp dir as bind volume for input/output data exchange
- Parse container exit code and handle non-zero as error
- Stream logs via Dockerode's `container.logs({ follow: true, stdout: true, stderr: true })`
- Add `dockerode` to package.json dependencies
