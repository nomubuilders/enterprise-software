# Task: End-to-end testing and graceful degradation

## Description
Comprehensive end-to-end testing of the Docker Container Node across all features, plus implementation of graceful degradation when Docker is unavailable. Verify the full workflow: drag node → configure → execute → capture output → audit log.

## Acceptance Criteria
- [ ] **E2E test**: Create workflow with Trigger → Docker Container → Output. Configure Docker node with alpine image, echo command. Run workflow. Verify output flows through to Output node
- [ ] **Allowlist test**: Attempt to execute non-allowlisted image → verify rejection with clear error
- [ ] **Resource test**: Configure container with CPU/RAM limits → verify limits applied via docker inspect
- [ ] **Timeout test**: Execute long-running container with short timeout → verify watchdog kills it
- [ ] **Air-gapped test**: Execute with networkMode=none → verify container has no network access
- [ ] **Audit test**: Run 3 different containers → verify 3 audit records in JSONL file
- [ ] **Graceful degradation**: When Docker daemon is unavailable: node shows "Docker Unavailable" status badge in gray, config panel shows warning message, execution attempt shows clear error ("Docker is not running. Please start Docker Desktop."), all other nodes and workflow features continue to work normally
- [ ] **Health check**: Docker status indicator in sidebar or status bar showing connection status
- [ ] **Theme test**: Docker node and config panel render correctly in both dark and light themes
- [ ] Zero TypeScript errors across all Docker-related files
- [ ] No orphaned containers after test suite completion

## Technical Details
- Health check: poll `/api/v1/docker/status` every 30 seconds, store result in dockerStore
- Graceful degradation: DockerContainerNode reads `dockerStore.isAvailable` and conditionally renders warning badge
- Config panel: disable "Execute" actions when Docker unavailable
- Docker status indicator: small colored dot (green=connected, gray=unavailable) near the Docker node in sidebar or in the bottom status bar of App.tsx
