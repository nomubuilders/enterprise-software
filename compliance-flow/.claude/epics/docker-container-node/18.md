# Task: Implement audit logging and ExecutionPanel integration

## Description
Add immutable audit logging for every container execution and integrate container execution events into the existing ExecutionPanel component so users see container logs alongside other workflow execution logs.

## Acceptance Criteria
- [ ] Every container execution writes an audit record to `data/audit/container-executions.jsonl` (append-only)
- [ ] Audit record includes: eventType, workflowId, workflowRunId, nodeId, userId, timestamp, image, imageSha256, command, resourceLimits, networkMode, exitCode, duration, outputSizeBytes
- [ ] Audit records written on both success and failure
- [ ] `src/components/modals/ExecutionPanel.tsx` updated to show container execution logs with a distinct icon (Container icon) and brand styling
- [ ] Container execution start/progress/complete events appear in the execution log stream
- [ ] Container errors (non-zero exit, timeout, security block) show in ExecutionPanel with appropriate error styling
- [ ] Audit log directory created automatically if missing

## Technical Details
- Audit logger: simple `fs.appendFileSync` to JSONL file (one JSON object per line, newline delimited)
- Create `data/audit/` directory in backend service initialization
- ExecutionPanel integration: add container-specific log entries to `currentExecution.logs` in workflowStore
- Use `Container` icon from lucide-react for container log entries (distinct from other node types)
- Log levels: `info` for start/complete, `error` for failures, `warn` for security blocks
